Feature Driven Development

LOREM IPSUM

O Feature Driven Development é uma metodologia ágil que organiza o desenvolvimento de software por progresso em recursos. Recursos esses que são similares ao user stories do Scrum (pequena unidade de desenvolvimento designada para completar uma meta dentro do produto). É extremamente prático para projetos complexos e de longa construção e times que procuram uma metodologia simples mas bem estruturada. Construção de lista de funcionalidades ou decomposição funcional se trata de uma análise de métodos onde processos complexos são divididos buscando facilitar a construção do projeto e criar uma lista das funcionalidades que deverão estar presentes no resultado final. Os processos são ordenados por prioridade de desenvolvimento, isso permite que cada parte seja analisada de forma independente e com isso é possível ter estimativas de tempo mais precisas e também identificar partes do projeto que podem contribuir para atrasos.

Planejar por funcionalidade (Planejamento incremental) : É a etapa em que um programa ou software é entregue em partes, que são denominados de incrementos independente do seu tamanho, porém, essas partes são entregues já em sua versão final. Reduzindo o número de falhas na solução entregue ao usuário com uma determinação de esforços em cada incremento para que ambas estejam funcionando corretamente. E com isso acaba criando um ambiente rápido e prático e com agilidade em solucionar os problemas que acontecem em cada etapa.
